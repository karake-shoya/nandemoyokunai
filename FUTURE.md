# FUTURE.md - なんでもよくない

将来的に実装したいアイデア・検討事項のメモ。
優先度・実現性は問わず、思いついたら追記していく。

## UX・機能アイデア

### 提案まわり
- [ ] メニューにタグ追加（ヘルシー・がっつり・時短・記念日など）
- [ ] 「今日の気分」入力から提案をカスタマイズ（疲れてる・元気・特別な日）
- [ ] 季節・気温に連動した提案（夏は冷麺、冬は鍋など）
- [ ] カテゴリ別の除外日数設定（カレーは14日、パスタは5日など）
- [ ] 「また食べたい」ボタンで除外を一時解除

### 記録まわり
- [ ] 料理の写真を撮って記録（Cloudflare R2保存）
- [ ] 満足度の記録（⭐1〜5）→ 提案精度の向上に活用
- [ ] カレンダービューで食事履歴を可視化

### コミュニケーション
- [ ] パートナー招待機能（招待リンクでペアリング）
- [ ] パートナーが自分の好みを直接入力できる
- [ ] 「今日はこれにしたよ」とパートナーに通知を送れる
- [ ] LINEで返答文章をそのまま送れる連携

### 通知
- [ ] PWAプッシュ通知（Phase 2）
- [ ] LINE通知オプション（Phase 3）
- [ ] 通知時間のカスタマイズ（17時固定→ユーザー設定）
- [ ] 「記録しないで」オプション（旅行・外泊時など）

## 既知の課題・懸念

- 楽天レシピAPIの利用規約：商用利用OK、ただし定期的に確認要
- iOS PWAプッシュ通知：Safari 16.4以降のみ対応、古い端末は非対応
- Cloudflare Workers edge runtime制約：一部Next.js APIが使えない可能性
- 「なんでもいい」問題の本質：アプリを使ってもらう習慣化が最大の課題
